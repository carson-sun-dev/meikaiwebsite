## 2. 关键技术决策

| 决策 | 选择 | 理由 |
|---|---|---|
| Excel 回流路径 | manual_areas.json(面积)+ manual_overrides.json(total/业态)双文件 | 面积是店主"凭印象 ±20%",和 type/total 同源同优先级但精度尺度不同;分两文件让 reviewer 一眼看出可靠度梯度 |
| 是否所有 23 个 project 全写进 overrides | 是,16+6 条全覆盖 | 店主真实付款金额与 parse_quotes 启发式抽到的总价差 5%-300%(合并单元格 + 多 sheet 重复合计 + 含税口径错);保守"只覆盖差异大的"会让分位数被脏数据带偏 |
| LangGraph 节点 streaming | `stream_writer({"kind": "delta", "text": ...})` + `astream(stream_mode=["updates", "custom"])` | LangGraph node 是 sync return state-diff,没法在 return 前 yield token;stream_writer 是 0.2.30+ 的 escape hatch,把 token-level 流式与 state-level 编排解耦 |
| LLM Router 输出后处理 | substring match("residential"/"office"/"storefront" in raw),fallback storefront | L0 模型对单词级输出格式遵从度高但偶尔有引号/解释,严格 JSON schema 不如子串匹配稳;美恺核心业务是门面房,unknown 兜底 storefront 最不会出洋相 |
| 报价工具 fallback 阈值 | `count >= 4` 且 `P90/P10 >= 1.3` 才用真桶 | 火锅店 n=2 实测 P10=2364 P90=2393(全 storefront 顶端),三档塌缩到几乎相等,用户体感"basic = premium"反而失去引导意义 |
| INDUSTRY_DEFAULT 兜底 | storefront ¥600-3000 / office ¥600-2400 / residential ¥500-2500 | 郑州 2026 行业经验粗值,storefront 跨度最大、residential 最小;source 字段如实标 `industry_default:*`,Guardrail 可据此放宽阈值 |
| `parse_quotes.normalize_header` 修了静默 bug | 剥中英文括号注解 | 6 个项目(住宅 + 学校 + 部分餐饮)有"金额（元）"列名,旧 normalize 只去空格,exact match 失败 → 0 subtotal → 空间分项 ETL 完全没数据。一行修复让 24 个新数据点入库 |
