## 2. 关键技术决策

| 决策 | 选择 | 理由 |
|---|---|---|
| LLM/Embedding 入口 | 火山方舟统一,1 份 key 多模型 | DeepSeek-V3.2 已托管在火山方舟,省一个账号;Doubao-embedding 下架后单独再选 |
| BM25 路径 | Jieba 客户端预 tokenize(不用 Qdrant 服务端 ICU) | 装修垂直术语 ICU 切碎(踢脚线/吊顶),Jieba + 装修词典整词命中 |
| tokenizer 治理 | `version_tag` 写进 Qdrant collection sentinel point,启动校验 | 防止词典升级导致 BM25 indices 漂移、检索静默退化 |
| ETL 打标策略 | manual_overrides > 规则法 > LLM 兜底 | 三层优先级清晰,manual 永远最权威 |
| 营销折扣实现 | 营销系数与历史数据**解耦** | 数据层永远是真实价(Guardrail 用),系数在输出层乘 |
| 报价展示形态 | 必须按"空间×工艺"拆分,严禁单一大数字 | 心理学:¥3 万 + ¥18 万 + ¥12 万 比 ¥53 万 友好得多 |
| ¥/㎡ 数据缺口 | 4 层数据回退:regex 自动扫 + 人工补 + LLM 兜底 + 业态默认值 | 装修业内没有面积字段规范,自动扫率 1/23,人工补是必经之路 |
